클로져란

클로저는 함수와 함수가 선언된 어휘적 환경의 조합이다.
클로저를 이해하려면 자바스크립트가 어떻게 변수의 유효범위를 지정하는지(Lexical scoping)를 먼저 이해해야 한다.

MDN 출처


스코프

자바스크립트의 스코프는 범위를 뜻한다.

function scope_a(){
	const consoleData = 'a';

<span class="kd">function</span> <span class="nx">scope_b</span><span class="p">(){</span>
	<span class="kd">const</span> <span class="nx">consoleData</span> <span class="o">=</span> <span class="dl">'</span><span class="s1">b</span><span class="dl">'</span><span class="p">;</span>

	<span class="nx">console</span><span class="p">.</span><span class="nx">log</span><span class="p">(</span><span class="nx">consoleData</span><span class="p">);</span> <span class="c1">// b</span>
<span class="p">}</span>

<span class="nx">scope_b</span><span class="p">();</span>
<span class="nx">console</span><span class="p">.</span><span class="nx">log</span><span class="p">(</span><span class="nx">consoleData</span><span class="p">);</span> <span class="c1">// a</span>



}


scope_a();


scope_a 함수에서 consoleData 변수에 값 ‘a’를 넣었고,
scope_b 함수에서 consoleData 변수에 값 ‘b’를 넣었다.

scope_b 함수를 실행하면서 consoleData의 변수 선언과 할당 과정이 있지만
scope_a 의 변수에는 전혀 영향이 없다.

즉, scope_a와 scope_b는 각각의 스코프를 가지고 있다.


스코프 체인

자바스크립트는 스코프 내에 참조할 수 있는 변수나 함수가 존재하지 않으면 바깥의 스코프에서 식별자 정보를 찾는다.

function scope_a(){
	const consoleData = 'a';

<span class="kd">function</span> <span class="nx">scope_b</span><span class="p">(){</span>
	<span class="nx">console</span><span class="p">.</span><span class="nx">log</span><span class="p">(</span><span class="nx">consoleData</span><span class="p">);</span> <span class="c1">// a</span>
<span class="p">}</span>

<span class="nx">scope_b</span><span class="p">();</span>



}


scope_a();


scope_b 함수 내부에 consoleData란 변수가 없지만 상위 스코프에서 식별자의 정보를 찾았다.


렉시컬 스코프 (Lexical scoping)

선언되는 시점의 바깥 스코프를 가리킨다는 것이다.

const consoleData = 1; // global

function scope_a() {
const consoleData = 10;
scope_b();
}


function scope_b() {
console.log(consoleData);
}


scope_a(); // ?
scope_b(); // ?


예측한 그대로 결과값이 나왔는가?

결과값은 1 / 1 이다.
scope_a를 통하여 scope_b를 호출할 때
scope_b는 호출하는 상위 스코프가 아닌
scope_b가 선언될 때 그 상위 스코프를 찾아서 출력하였다.

그래서 클로져는 ?

클로져는 어떤 함수(outer) 내부에 선언된 함수(inner)가 바깥 함수(outer)의 지역변수(outerVariable)를 참조하는 것이 함수(outer)가 종료된 이후에도 계속 유지되는 현상을 말한다.

const outer = () => {
	const outerVariable = 'outer!'; // 1. 바깥 함수 outer의 스코프에 변수선언

<span class="kd">const</span> <span class="nx">inner</span> <span class="o">=</span> <span class="p">()</span> <span class="o">=&gt;</span> <span class="p">{</span>
	<span class="nx">console</span><span class="p">.</span><span class="nx">log</span><span class="p">(</span><span class="nx">outerVariable</span><span class="p">);</span> <span class="c1">// 2. 내부 함수 inner의 스코프에서 스코프체인을 타고 바깥 함수 스코프의 변수 참조</span>
<span class="p">};</span>

<span class="k">return</span> <span class="nx">inner</span><span class="p">;</span> <span class="c1">// 3. inner를 바깥으로 반환</span>



};


const closureFn = outer(); // 4.  fano에 inner함수의 주소값이 저장됨


closureFn(); // 5. outer함수 호출은 종료가 되어서 스코프가 사라져야 하지만 outerVariable은 여전히 잘 참조된다.


즉, 클로져는 어떤 함수(outer) 내부에 선언된 함수(inner)가
바깥 함수(outer)의 지역변수(outerVariable)를 참조하는 것이
함수(outer)가 종료된 이후에도 계속 유지되는 현상을 말한다.

클로져를 사용한 예제

	const calculator = () => {
		let value = 0;

	<span class="k">return</span> <span class="p">{</span>
		<span class="nx">increase</span><span class="p">()</span> <span class="p">{</span>
			<span class="nx">console</span><span class="p">.</span><span class="nx">log</span><span class="p">(</span><span class="o">++</span><span class="nx">value</span><span class="p">);</span>
		<span class="p">},</span>
		<span class="nx">decrease</span><span class="p">()</span> <span class="p">{</span>
			<span class="nx">console</span><span class="p">.</span><span class="nx">log</span><span class="p">(</span><span class="o">--</span><span class="nx">value</span><span class="p">);</span>
		<span class="p">}</span>
	<span class="p">}</span>
<span class="p">};</span>

<span class="kd">const</span> <span class="nx">a_counter</span> <span class="o">=</span> <span class="nx">calculator</span><span class="p">();</span>
<span class="kd">const</span> <span class="nx">b_counter</span> <span class="o">=</span> <span class="nx">calculator</span><span class="p">();</span>


<span class="nx">a_counter</span><span class="p">.</span><span class="nx">increase</span><span class="p">();</span> <span class="c1">// 1</span>
<span class="nx">a_counter</span><span class="p">.</span><span class="nx">increase</span><span class="p">();</span> <span class="c1">// 2</span>
<span class="nx">a_counter</span><span class="p">.</span><span class="nx">decrease</span><span class="p">();</span> <span class="c1">// 1</span>
<span class="nx">a_counter</span><span class="p">.</span><span class="nx">increase</span><span class="p">();</span> <span class="c1">// 2</span>
<span class="nx">b_counter</span><span class="p">.</span><span class="nx">decrease</span><span class="p">();</span> <span class="c1">// -1</span>
